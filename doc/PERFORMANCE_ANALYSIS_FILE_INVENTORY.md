### 2. test_scripts/ - 测试脚本目录

#### 2.1 performance_test.py
- **类型**: Python 测试脚本
- **内容**: 整体性能测试
- **功能**:
  - 测试应用程序的整体帧率表现
  - 测量逻辑更新和绘制操作的耗时
  - 分析时间分配和性能瓶颈
  - 生成基础性能报告
- **主要测试指标**:
  - 平均/最低/最高 FPS
  - 平均帧时间
  - 逻辑更新耗时
  - 绘制操作耗时
- **依赖**:
  - `pygame` - 图形库
  - `time` - 时间测量
  - `Core.*` - 核心模块
  - `Scene` - 场景管理
- **输出**: 控制台性能报告
- **运行时间**: 约10秒
- **关联文件**:
  - 输入: `../src/Main.py`, `../src/Scene.py`
  - 输出: 控制台报告
- **更新时间**: 2026-03-16

#### 2.2 clustering_analysis.py
- **类型**: Python 测试脚本
- **内容**: 聚类算法详细性能分析
- **功能**:
  - 测试不同聚类间隔的性能影响
  - 分析不同 Actor 数量的聚类性能
  - 评估 DBSCAN 算法的复杂度
  - 提供聚类优化建议
- **主要测试指标**:
  - 不同聚类间隔的耗时
  - 不同 Actor 数量的聚类时间
  - 聚类算法复杂度分析
  - 每秒聚类开销
- **依赖**:
  - `pygame` - 图形库
  - `time` - 时间测量
  - `ActorClustering` - 聚类算法
- **输出**: 控制台聚类性能报告
- **运行时间**: 约30-60秒
- **关联文件**:
  - 输入: `../src/clustering/ActorClustering.py`
  - 输出: 控制台报告
- **更新时间**: 2026-03-16

#### 2.3 rendering_analysis.py
- **类型**: Python 测试脚本
- **内容**: 渲染性能详细分析
- **功能**:
  - 测试单个和批量绘制性能
  - 对比不同绘制方法的效率
  - 分析显示翻转和屏幕清除开销
  - 提供渲染优化建议
- **主要测试指标**:
  - 单个 Actor 绘制耗时
  - 批量绘制性能
  - 不同绘制方法对比
  - 显示翻转耗时
  - 屏幕清除耗时
- **依赖**:
  - `pygame` - 图形库
  - `time` - 时间测量
  - `Graph` - 绘制工具
- **输出**: 控制台渲染性能报告
- **运行时间**: 约20-40秒
- **关联文件**:
  - 输入: `../src/Core/Graph.py`
  - 输出: 控制台报告
- **更新时间**: 2026-03-16

#### 2.4 memory_analysis.py
- **类型**: Python 测试脚本
- **内容**: 内存使用和泄漏分析
- **功能**:
  - 监控内存使用变化
  - 检测内存泄漏
  - 分析对象生命周期
  - 评估垃圾回收效率
- **主要测试指标**:
  - 内存使用增长
  - 峰值内存占用
  - 对象创建/销毁内存
  - 内存分配热点
- **依赖**:
  - `pygame` - 图形库
  - `tracemalloc` - 内存跟踪
  - `gc` - 垃圾回收
- **输出**: 控制台内存分析报告
- **运行时间**: 约30-60秒
- **关联文件**:
  - 输入: `../src/Actor.py`, `../src/Grid.py`
  - 输出: 控制台报告
- **更新时间**: 2026-03-16

## 📊 数据流向

### 测试执行流程
1. **准备阶段**: 读取配置和源代码
2. **执行阶段**: 运行性能测试脚本
3. **数据收集**: 收集性能数据
4. **分析阶段**: 生成分析报告
5. **保存阶段**: 保存原始数据和报告

- **测试脚本** → **输出数据** (控制台/数据文件)
- **数据文件** → **分析报告**
- **任务状态** → **所有文件**

## 🎯 文件命名规范

### 测试脚本
- 格式: `<功能>_analysis.py` 或 `<功能>_test.py`
- 示例: `performance_test.py`, `clustering_analysis.py`

### 报告文件
- 格式: `<主题>报告.md` 或 `<主题>_report.md`
- 示例: `性能分析报告.md`

### 数据文件
- 格式: `<类型>_<时间戳>.<扩展名>`
- 示例: `Profile_20260301231304.prof`

### 文档文件
- 格式: `<主题>.md`
- 示例: `task_status.md`, `README.md`
